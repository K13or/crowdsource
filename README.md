# Народный перевод Guild Wars 2 на русский — как участвовать

Это краудсорс-перевод игрового текста Guild Wars 2 для локализатора **GlyphCore**
(https://github.com/nourlie/GlyphCore). Весь непереведённый текст нарезан на **батчи**
по ~500 строк. Берёшь батч, переводишь колонку `translate`, сдаёшь — и он попадает в игру.

## Как взять батч («забиндить»)

1. Открой **[CLAIMS.md](CLAIMS.md)** — доску захвата.
2. Найди свободный батч (🟢), впиши свой ник в колонку «Кто взял» и смени статус на 🔨 (в работе).
   *(через Pull Request, комментарий или как договоритесь в чате проекта.)*
3. Скачай свой файл из папки батчей, напр. `ui/ui_042.csv` или `items/items_010.csv`.
4. Переведи и сдай (см. ниже). Смени статус на ✅.

## Как переводить

Файл батча — это CSV с двумя колонками:

```csv
english,translate
Achievement Complete,
A cache key is required to open this chest.,
```

- **Заполняй только вторую колонку** (`translate`). **Первую (`english`) не меняй.**
- Строки, которые не знаешь как перевести, оставляй пустыми — их возьмёт кто-то другой.
- Сохраняй файл в кодировке **UTF-8**.

### ⚠️ Обязательные правила (иначе строка сломается в игре)

1. **Сохраняй плейсхолдеры дословно:** `%str1%`, `%num1%`, `%str2%` … — игра подставляет
   в них имена/числа. Их набор в переводе должен совпадать с оригиналом.
   `%str1% has joined the party.` → `%str1% присоединился к группе.`
2. **Сохраняй теги:** `<c=@flavor>…</c>`, `<c=#ff8c00>…</c>`, `<br>`, `[null]`,
   `[lbracket]` / `[rbracket]`. Переводи только текст внутри, сами теги оставляй как есть.
3. **Склонения по числу.** Английское `[s]` (множественное) в русском передаётся **тремя
   формами** через `|` в скобках — движок сам выберет по числу:
   `[форма_для_1 | форма_для_2-4 | форма_для_5+]`
   Пример: `%num1% Snowflake[s]` → `%num1% снежин[ка|ки|ок]`.
   Две формы `[ый|ые]` — для 1 / для остальных.
4. **Имена и названия.** Имена персонажей и локаций **автоматически** превращаются в кириллицу
   на лету (слой `pn_*.csv`). Поэтому известные имена можно оставлять латиницей —
   движок сам подставит русскую форму. Если имени нет в глоссарии, впиши русскую форму сам.
   Сверяйся с файлами `pn_names.csv`, `pn_terms.csv`, `pn_world_map.csv` в папке `cyrillic\`.

### Стиль

- Естественный литературный русский, а не подстрочник. Сохраняй тон (грубый чарр — грубо,
  учёный асура — заумно, шутка — шуткой).
- **Термины — по канону.** Боевые/механические термины (профессии, благодати, состояния, контроль,
  комбо, атрибуты) бери из **[GLOSSARY.md](GLOSSARY.md)** и с русской вики
  **https://ru.gw2skills.net/wiki** (эталон). Имена/локации — из глоссариев `pn_*`. Единообразие важнее личных предпочтений.
- Обращение к игроку — на «вы» в интерфейсе/подсказках; в живых репликах — по ситуации.

## Что НЕ переводить

Технические строки (чистые коды, ссылки, числа) в батчи не попали — их отфильтровали.
Если в батче попалась строка на другом языке (напр. немецком) — оставь пустой.

## Как сдать батч

Сдай заполненный CSV так, как договорено в чате проекта: **Pull Request** в репозиторий,
загрузка в общую папку или в Discord. Мейнтейнер вольёт его в игру скриптом
**[merge_back.py](merge_back.py)**:

```
python merge_back.py <путь_к_папке_cyrillic> <папка_или_файл_с_готовыми_батчами>
```

Скрипт сам находит строки во всех игровых файлах, заполняет их переводом, проверяет
плейсхолдеры и не трогает уже переведённое.

## Отключаемые словари названий

Названия предметов вынесены из общего файла в отдельный словарь
**`dict_items_names.csv`** (описания остались в `dict_items.csv`). Прокси загружает
только файлы с префиксом `dict_`, поэтому любой словарь можно временно выключить,
просто переименовав его — названия в игре снова станут английскими (удобно для
вики, торговли и LFG):

```
dict_items_names.csv  →  off_dict_items_names.csv   (названия предметов — EN)
off_dict_items_names.csv  →  dict_items_names.csv   (обратно — RU)
```

То же самое работает для `dict_skins.csv` (облики), `dict_maps.csv` (локации),
`dict_colors.csv` (краски), `dict_minis.csv` (миниатюры) и любого другого словаря.
Имена персонажей и мест переключаются отдельно — опцией `translate_proper_nouns`
в `config.ini` (слой `pn_*.csv`).

`merge_back.py` вливает переводы и в выключенные (`off_dict_*.csv`) словари,
так что ничего не теряется. Переводчикам батчей это ничего не меняет:
переводим как раньше, скрипт сам кладёт строки в нужный файл.

**Правило для названий:** все названия (предметов, оружия, брони — включая
шаблоны линковки `%str1%%str2%Название%str3%%str4%`) живут в `dict_items_names.csv`
или в профильных словарях (`dict_skins`, `dict_maps`…), а НЕ в `cyrillic_strings.csv` —
иначе они не будут отключаться. Новые отдельные словари не создаём — добавляем
строки в уже существующие.
